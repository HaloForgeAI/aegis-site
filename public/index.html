Owner-first architecture

Core owns state. Apps and nodes bring the interface and hands.

Aegis Core keeps sessions, tasks, approvals, memory, tools, channels, provider traces, and evidence. Native apps are the daily user experience. Local Gateway and Native Nodes execute explicit payloads on computers and devices instead of guessing from prose.

Quick install

Choose the path that matches your setup.

Aegis is moving to native application installs first. Start with the app for your platform, then choose how your devices reach Aegis Core.

1

macOS

Aegis-<version>-macos-arm64.dmg

Install Aegis.app from the DMG. The desktop app manages local Core, Local Gateway, nodes, Chat, Workbench, and runtime status.

2

Windows

Aegis-<version>-windows-x64.msix

Use the signed MSIX. Portable zip builds are dev diagnostics, not public install assets; MSI is an enterprise follow-up.

3

Mobile

iOS/TestFlight · IPA · Android APK/AAB

Pair iPhone, iPad, or Android as daily Chat and approval surfaces, and as Mobile Native Nodes where platform permissions allow.

Connection choices

  • Local-only for one computer
  • LAN for trusted home or lab networks
  • Tailscale Serve for private owner-device access
  • Cloudflare Tunnel for stable HTTPS hostnames and webhooks
  • Public VPS only with Aegis auth and scoped tokens

Architecture

Aegis Manager plans. Workers and nodes execute explicit calls.

Ordinary chat and channel messages go through Aegis Manager and a real provider. Workers, Local Gateway, and Native Nodes receive structured tool or task payloads, execute them, and record evidence. Channels are useful surfaces; they are not the product's hidden parser.
